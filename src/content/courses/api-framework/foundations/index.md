Before a single line of routing code, we're clear on *why* this thin layer is worth building — to understand the framework you'll run in production, and to own the conventions every team ends up writing for itself. Then we design the core abstraction — a handler signature and a request context — that the rest of the course builds on. Get these two types right and everything downstream falls into place.
