*Most infrastructure pain is architectural before it is operational.* Teams feel it as drift, repeated copy-paste, brittle promotion flows, and modules nobody wants to touch. Terraform makes those problems visible quickly, but it does not solve them for you.

This catalog is a Terraform-specific reference for designing modules, shaping state boundaries, and organizing delivery workflows that can survive real teams and real change. Start with the [Design Philosophy](/terraform/philosophy) pages if you want the mental model first, then use the catalog to choose patterns that match the pressure your platform is under.
