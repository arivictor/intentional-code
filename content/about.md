# About

I'm Ari Victor. I build production Go in financial services, where the cost of the wrong structural call isn't abstract — it's a page at 3am, a migration that can't be rolled back, an audit that finds the seam you didn't draw. Most of what I know about architecture I learned from changes that turned out harder than they should have been.

Intentional Code exists because the resource I wanted didn't. The pattern catalogues I found showed me the code and skipped the call: *when* to reach for a pattern, what it costs, and when reaching for it is itself the mistake. So every page here carries the decision, not just the implementation — opinionated enough to disagree with, specific enough to be wrong about.

If you think one of my calls is wrong, you're using the site exactly as intended. [Tenet #10](/go/philosophy/disagree-with-reason) is that disagreement is welcome, as long as you can name the reason — and the whole thing is open source, so you can argue with it in public: [github.com/arivictor/intentional-code](https://github.com/arivictor/intentional-code).
